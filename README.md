# web-site

A static HTML/CSS/JS website template with HTMX for dynamic interactions. No build step, no dependencies. Created with [dade](https://github.com/theydontwantyoutovibecode/made-in-dade).

## Prerequisites

None. The template uses plain HTML, CSS, and JavaScript.

## Project Structure

```
.
├── index.html          # Main page
├── css/
│   ├── input.css       # Tailwind CSS source (with @tailwind directives)
│   └── style.css       # Compiled CSS (auto-generated)
├── js/
│   └── main.js         # JavaScript
├── lib/
│   └── htmx.min.js     # HTMX library (vendored)
├── partials/
│   ├── hello.html      # Example HTMX partial
│   └── more.html       # Example HTMX partial
├── tailwind.config.js  # Tailwind CSS configuration
├── dade.toml           # Template manifest
├── AGENTS.md           # AI agent instructions
└── .read-only/
    └── manifest.txt    # Reference repos for AI context
```

## Usage

### Development

```bash
dade dev
```

Starts a built-in static file server with Tailwind CSS watch mode. The project is available at `https://<project-name>.<hostname>.local`.

The dev command:
1. Starts Tailwind CSS in watch mode to compile `css/input.css` to `css/style.css`
2. Starts the static file server on the project's assigned port
3. Watches for CSS changes and auto-recompiles

### CSS Structure

- `css/input.css` — Your Tailwind CSS source file with `@tailwind` directives
- `css/style.css` — Compiled CSS (auto-generated by Tailwind in watch mode)
- `tailwind.config.js` — Tailwind configuration

You can add custom CSS in `input.css` below the Tailwind directives.

### Watch Mode

During development, Tailwind CSS runs in watch mode:

```bash
tailwindcss -i css/input.css -o css/style.css --watch
```

This automatically recompiles your CSS whenever you make changes to:
- `css/input.css`
- `index.html`
- Any files in `partials/` or `js/`

The compiled `css/style.css` updates instantly without manual intervention.

### Sharing

```bash
dade share
```

Starts the static server and creates a Cloudflare tunnel for public access. Requires `cloudflared`.

### Not Applicable

- `dade project start` — No production server mode. The static file server is the same in dev and production.
- `dade build` — Nothing to compile (Tailwind compiles automatically in dev).

## Tailwind CSS

The template uses Tailwind CSS for styling via the standalone `tailwindcss` binary. No Node.js or npm required.

### How it works

1. **Input file**: `css/input.css` contains Tailwind directives:
   ```css
   @tailwind base;
   @tailwind components;
   @tailwind utilities;
   ```

2. **Compilation**: The `tailwindcss` binary compiles `input.css` to `style.css`

3. **Watch mode**: During development, Tailwind watches for changes and auto-recompiles

### Customization

Edit `tailwind.config.js` to:
- Add custom colors, fonts, or breakpoints
- Extend the theme
- Add plugins

Add custom CSS below the Tailwind directives in `input.css`.

### Using Tailwind

Use Tailwind's utility classes directly in your HTML:

```html
<button class="bg-blue-600 hover:bg-blue-700 text-white font-medium py-2 px-4 rounded-lg">
  Click Me
</button>
```

## HTMX

HTMX is loaded via `lib/htmx.min.js` (vendored, no CDN). Use HTMX attributes on HTML elements to load content dynamically.

The `partials/` directory contains HTML fragments. HTMX requests load these fragments and swaps them into the page without full page reloads.

Example:

```html
<button hx-get="/partials/hello.html" hx-swap="innerHTML" hx-target="#content">
  Load
</button>
<div id="content"></div>
```

## Deployment

The project is a collection of static files. Deploy to any static hosting service:

- Copy the entire project directory to a web server
- Upload to GitHub Pages, Netlify, Vercel, or Cloudflare Pages
- Serve with any HTTP server (nginx, Apache, Caddy)

No build step is required.
